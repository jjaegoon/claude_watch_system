## 설계 결정

### 테스트 격리 전략

- **JWT 상태 격리**: `__resetForTest()` (tokenService.ts 내보낸 테스트 전용 함수)를 각 describe의 `beforeAll`에서 호출
- **rate limit 격리**: `X-Forwarded-For` 헤더로 IP 분리 (describe별 고유 IP 할당). `IP_RATE_LIMIT = '10.0.99.1'`은 rate limit 테스트 전용
- **실 DB 사용**: `dev.db` (seeded — 5 users + 7 assets). `health.test.ts`와 동일 패턴

### UUID 검증 함정 (자율 발견)

`GET /assets/does-not-exist-00000000` → 400 (not 404). 이유: `assetIdParamSchema`가 `z.string().uuid()` 검증을 선행. UUID 형식이 아니면 400 INVALID_INPUT 먼저 반환.

**처방**: 테스트에서 valid UUID format 사용 (`'00000000-0000-0000-0000-000000000001'`). 존재하지 않으면 404.
