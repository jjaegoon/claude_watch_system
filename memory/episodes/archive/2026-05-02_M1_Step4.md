## 함정 ①: FTS5 external content DELETE trigger 패턴 (gotcha #14)

**증상**: migration 010 최초 작성 시 DELETE trigger에 `DELETE FROM assets_fts WHERE rowid = old.rowid` 사용. 자산 삭제 후에도 FTS MATCH에 계속 매칭됨.

**원인**: FTS5 external content 테이블은 DML 직접 실행 불가. `INSERT INTO assets_fts(assets_fts, rowid, ...) VALUES('delete', ...)` 특수 명령으로만 인덱스 갱신 가능.

**해결**: DELETE trigger → `INSERT ... VALUES('delete', old.rowid, ...)` 패턴. UPDATE trigger → delete-old 후 insert-new 두 단계.

### 함정 ②: C-1 FTS5 `-` negation (gotcha 후보)

**증상**: `buildFts5Query('code-review')` → `'code-review*'`. SQLite FTS5가 `-review*`를 `NOT review*`로 해석 → "fts5: no such column: review" 오류.

**해결**: FTS5_RESERVED에 `-` 추가. `code-review` → `'code* AND review*'` (T-27 최종 spec과 일치).

**영향**: searchService.test.ts case 3 기대값 `'code-review*'` → `'code* AND review*'`로 수정.
