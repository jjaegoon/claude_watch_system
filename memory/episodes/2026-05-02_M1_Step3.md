## 새 발견 / 함정

### 함정 ①: JWT deterministic — 동일 payload + 동일 iat 호출 시 토큰 충돌

**증상**: 단위 테스트 "blacklistAllByUser 후 새 refresh_token은 정상 발급" 실패. 도난 감지 → blacklistAllByUser → 즉시 재로그인 시 발급된 새 refresh_token이 이전 token과 동일한 string → isBlacklisted=true 오판.

**원인**: jsonwebtoken은 deterministic — 같은 payload + 같은 secret + 같은 iat(초 단위) → 동일 token. 마이크로초 단위 호출 시 충돌.

**해결**: `jwt.sign(...)` 옵션에 `jwtid: crypto.randomUUID()` 추가. 매 토큰마다 unique jti claim으로 string 충돌 차단.

**향후**: gotcha #10 후보. JWT 라이브러리 일반 함정 — 빈도 따라 등재 결정.

### 함정 ②: bcrypt prebuilt 자동 다운로드 (Linux glibc 의존)

**증상**: macOS arm64에선 prebuilt napi-v3 정상 다운로드 (`darwin-arm64-unknown.tar.gz`).

**미래 위험**: D+29 GitHub Actions Linux glibc 빌드 시 prebuilt 호환 안 될 수 있음. node-pre-gyp가 source build로 fallback — gcc/python 필요. 실패 시 `bcryptjs`(pure JS, 약 2배 느림) 교체 — Plan 리스크 #8 정합. **T-39 후보**.
