## 핵심 설계 결정

### T-16 State Machine (assetStatusService.ts)

```
draft      → in_review  : author OR reviewer+
in_review  → approved   : reviewer+ AND author≠userId (T-34: system_user exempt)
in_review  → draft      : reviewer+ AND reason_code 필수 (R-01~R-12)
approved   → deprecated : admin
deprecated → approved   : admin
```

### T-34 system_user 면제

```ts
if (authorId === userId && role !== 'system_user') {
  return { ok: false, code: 'FORBIDDEN', message: 'self-review 금지 (T-16)' }
}
```

### T-15 webhookWorker ORDER BY gotcha (#18)

pending job 선택 쿼리에 `ORDER BY created_at, rowid LIMIT 1` — 동일 초 삽입 시 결정적 순서 보장.

### CSRF bypass

`csrfGuard.ts`에 이미 `/assets/sync` 예외 존재 (Stage 2 pre-config). `webhookRoute`는 `assetsRoute` BEFORE 마운트 — requireAuth 미적용.

## 해결한 문제들

### assetWriteService.test.ts typecheck 오류

`CreateAssetInput` = Zod output type with `.default([])` → `tags: string[]` required. 6개 테스트 입력에 `tags: [], typeFields: {}` 추가.

### assetStatusService.test.ts change_note 비결정 (gotcha #18)

`ORDER BY created_at DESC LIMIT 1` → `ORDER BY rowid DESC LIMIT 1` 변경. 동일 초 내 두 INSERT 모두 같은 created_at 값.

### webhookWorker.test.ts cross-test 오염

`beforeEach(() => db.exec('DELETE FROM webhook_jobs'))` 추가. 이전 테스트 잔류 job이 다음 테스트의 processNextJob에 선택되는 문제.

### webhook-jobs e2e: worker가 다른 job 처리

drain loop 도입: `while (processNextJob(sqlite) && drained++ < 10)` — 이전 테스트의 pending job 모두 소진 후 특정 jobId 검증.
